TUTORIAL DI KERNEL LAB

Questo sito offre un tutorial su come usare Kernel Lab e descrive i suoi utilizzi pratici.

Kernel Lab è un progetto sviluppato per analizzare, diagnosticare e comprendere gli errori e i messaggi generati dal kernel nei sistemi operativi Linux e UNIX-like, tra cui Arch Linux, Ubuntu, Debian, Fedora, Kali Linux e molti altri.

Il kernel rappresenta il cuore del sistema operativo: gestisce le risorse hardware, la memoria, i processi e garantisce che tutto il sistema funzioni in modo coordinato. Quando si verificano malfunzionamenti, crash o comportamenti anomali, le informazioni più importanti per capire l'origine del problema si trovano nei log del kernel. Tuttavia, questi log sono spesso lunghi, complessi e non immediatamente comprensibili, soprattutto per chi non ha molta esperienza.

Kernel Lab nasce proprio per rendere questa analisi più semplice, chiara e accessibile. Il programma consente di leggere i log del kernel in modo ordinato e filtrato, individuare automaticamente eventuali crash o anomalie, e visualizzare gli eventi significativi attraverso una comoda timeline. Il tutto avviene tramite un'interfaccia testuale interattiva (TUI) che garantisce velocità e facilità d'uso, anche in ambienti privi di interfaccia grafica.

Che tu sia un amministratore di sistema, uno sviluppatore o uno studente curioso di capire come funziona il cuore di un sistema Linux, Kernel Lab ti aiuta a monitorare la stabilità e la salute del tuo sistema operativo in modo efficace e centralizzato.

Ma che cos'è di preciso una TUI e perché usarla?

Una TUI, o Text-based User Interface, è un'interfaccia utente che funziona tramite testo all'interno di un terminale o console. A differenza delle GUI (interfacce grafiche), le TUI utilizzano caratteri e simboli per creare menu, finestre, e interazioni, mantenendo però la leggerezza e la rapidità tipiche delle applicazioni a riga di comando.

Usare una TUI può essere molto vantaggioso in diversi contesti: consente di avere un ambiente interattivo e user-friendly anche in assenza di un sistema grafico, è ideale per amministratori di sistema e sviluppatori che lavorano spesso da remoto via SSH(Secure Socket Shell), e permette di monitorare e controllare sistemi complessi in modo efficiente senza appesantire le risorse.

Nel caso di Kernel Lab, la TUI rappresenta il modo perfetto per offrire agli utenti un'interfaccia semplice, immediata e integrata con la shell, che consente di analizzare i log del kernel e identificare crash in modo rapido senza dover uscire dall'ambiente di lavoro oltre ad avere le normali funzioni di una shell di linux.

Nei vari comandi noi analizzeremo i piu semplici nonchè in piu importanti, ovvero:

  • ls
  • cd
  • touch
  • pwd
  • echo
  • rm
  • cp
  • mov
  • grep

e altri.

COMANDI DI LINUX

Comando A cosa serve Descrizione
ls Lista i file Visualizza l'elenco dei file e delle directory presenti nella directory corrente. Può essere arricchito con opzioni, ad esempio `ls -l` mostra dettagli come permessi, proprietario, dimensioni e data di modifica; `ls -a` include anche i file nascosti.
cd Cambia directory Cambia la directory corrente. Può essere usato con percorsi assoluti o relativi. Ad esempio `cd /home/utente` va in quella directory, mentre `cd ..` torna alla cartella superiore.
pwd Mostra percorso Mostra il percorso assoluto della directory in cui ti trovi attualmente. È utile per sapere dove ci si trova nel filesystem, soprattutto durante l'uso di script o sessioni remote.
mkdir Crea una cartella Crea una o più directory. Con l'opzione `-p` può creare anche directory annidate, es. `mkdir -p a/b/c` creerà tutte le sottodirectory se non esistono.
rm Rimuove file o cartelle Rimuove file o directory. Senza opzioni rimuove solo file singoli; con `-r` (ricorsivo) può eliminare intere directory. L'opzione `-f` forza la rimozione senza chiedere conferma. Attenzione: le rimozioni sono permanenti.
cp Copia file Copia file e directory. `cp file1 file2` copia `file1` in `file2`. L'opzione `-r` permette di copiare directory intere. Altre opzioni utili includono `-u` (solo file più recenti) e `-i` (chiede conferma prima di sovrascrivere).
>mv Sposta o rinomina Sposta file o directory in una nuova posizione o cambia loro nome. Ad esempio, `mv file.txt backup/` sposta il file in una cartella; `mv vecchio.txt nuovo.txt` lo rinomina. È anche usato per organizzare file rapidamente.
touch Crea file vuoti Crea uno o più file vuoti, o aggiorna il timestamp di modifica di un file esistente. Utile per creare rapidamente segnaposto o file di log.
cat Visualizza contenuto Concatena e visualizza il contenuto di uno o più file sul terminale. Ad esempio `cat file.txt` mostra il contenuto di quel file. È utile anche per redirigere contenuti (`cat file1 file2 > unito.txt`).
echo Stampa testo Stampa una stringa sul terminale. Usato spesso negli script per mostrare messaggi o scrivere contenuti in un file (`echo "ciao" > saluto.txt`). Supporta variabili e sequenze speciali (`\n`, `\t`).
man Manuale comandi Mostra il manuale di un comando, fornendo descrizioni dettagliate, opzioni e utilizzi. Es. `man ls` spiega tutto sul comando `ls`. Utilissimo per apprendere le funzionalità avanzate di ogni strumento.
grep Cerca nel testo Cerca stringhe o espressioni regolari all'interno di file o output. Es. `grep "ciao" file.txt` trova tutte le righe contenenti "ciao". Potente per filtrare dati, log, e combinabile con altri comandi via pipe (`|`).
chmod Modifica permessi Cambia i permessi di accesso a file o directory. Può essere usato in modo simbolico (`chmod u+x file`) o numerico (`chmod 755 file`). Utile per garantire che solo determinati utenti possano leggere, scrivere o eseguire file.
ps Visualizza processi Mostra l'elenco dei processi attivi. Con `ps aux` fornisce informazioni dettagliate su tutti i processi dell'utente e di sistema, inclusi PID, utilizzo della CPU/memoria, stato e comando di avvio.
kill Termina processi Termina un processo specificando il suo PID. Es. `kill 1234` termina il processo con PID 1234. Con `-9` si forza la chiusura immediata. È usato in caso di processi bloccati o che consumano troppe risorse.
top Monitora risorse Visualizza in tempo reale l'utilizzo della CPU, della memoria e altri dettagli sui processi in esecuzione. È utile per diagnosticare problemi di performance o identificare processi che consumano troppe risorse.
htop Monitor avanzato Versione migliorata e interattiva di `top`, con interfaccia a colori, ordinamenti personalizzabili e comandi di gestione dei processi più intuitivi. Deve essere installato separatamente sulla maggior parte dei sistemi.
find Trova file Permette di cercare file e directory in modo ricorsivo, in base a criteri come nome, tipo, dimensione o data. Esempio: `find . -name "*.txt"` cerca tutti i file `.txt` nella directory corrente e nelle sottocartelle.
tar Comprimi/estrai Crea archivi compressi o estrae file da archivi. Usato comunemente con gzip (`.tar.gz`). Esempio: `tar -czf archivio.tar.gz cartella/` crea un archivio compresso; `tar -xzf archivio.tar.gz` lo estrae.
df Spazio disco Mostra lo spazio disponibile e utilizzato su ciascun filesystem. L'opzione `-h` rende l'output leggibile (`df -h`) mostrando dimensioni in MB/GB. Utile per controllare se un disco è quasi pieno.
du Dimensione file/cartelle Riporta l'utilizzo dello spazio su disco da parte di file e directory. `du -sh cartella/` restituisce la dimensione totale della cartella in formato leggibile. Molto utile per capire cosa occupa spazio.
nano Editor di testo Editor di testo semplice da terminale, adatto a modifiche rapide. Offre comandi visuali per salvare, uscire, cercare, ecc. Ad esempio: `nano file.txt` apre il file per modificarlo.
vim Editor avanzato Editor di testo molto potente ma con curva di apprendimento più ripida. Modalità diverse (comando, inserimento), estendibile con plugin, ideale per programmatori. Esempio: `vim script.sh` apre il file per editing.
whoami Mostra utente attuale Restituisce il nome dell'utente attualmente connesso alla sessione terminale. Utile in script o su sistemi multiutente per confermare i privilegi in uso.
uptime Tempo acceso Indica da quanto tempo è acceso il sistema, insieme al carico medio della CPU. Utile per monitorare la stabilità e il carico generale della macchina.
history Comandi usati Mostra la cronologia dei comandi eseguiti nel terminale corrente. Ogni comando ha un numero associato, utile per richiamarlo rapidamente con `!numero`.
clear Pulisce lo schermo Rimuove tutto il testo visualizzato nel terminale corrente, rendendolo vuoto e pronto per nuovi comandi. Non cancella la cronologia, solo l'aspetto visivo.
alias Comandi personalizzati Permette di creare scorciatoie per comandi lunghi. Es. `alias ll="ls -la"` crea un nuovo comando `ll` per elencare file in modo dettagliato. Utile per semplificare operazioni ripetitive.
ping Controlla connessione Invia pacchetti ICMP a un host per verificarne la raggiungibilità e misurare il tempo di risposta. Esempio: `ping google.com` mostra se il sito è raggiungibile e quanto tempo impiega a rispondere. Utile per diagnosticare problemi di rete.
curl Scarica contenuti Strumento per trasferire dati da o verso un server usando protocolli come HTTP, FTP, ecc. Esempio: `curl https://example.com` scarica il contenuto della pagina. Può essere usato anche per inviare dati (API REST).
wget Scarica file Utile per scaricare file da internet via riga di comando. Esempio: `wget https://example.com/file.zip` salva il file nella directory corrente. Può scaricare ricorsivamente interi siti web o directory.
traceroute Traccia percorso Mostra il percorso (hop) seguito dai pacchetti per raggiungere un host remoto. Aiuta a individuare dove si verificano rallentamenti o problemi di connessione lungo la rete.
netstat Info sulla rete Mostra connessioni di rete attive, porte in ascolto, tabelle di routing e altro. Utile per capire se un servizio è in ascolto su una porta o se ci sono connessioni sospette.
ss Socket di rete Alternativa moderna e più veloce a `netstat`. Mostra socket attivi, connessioni TCP/UDP, porte in ascolto e statistiche dettagliate sul traffico di rete.
apt Gestisce pacchetti Gestore di pacchetti per sistemi basati su Debian (es. Ubuntu). Usato per installare, aggiornare o rimuovere software. Es. `apt install nomepacchetto` installa un programma dal repository ufficiale.
yum Pacchetti RedHat Gestore di pacchetti per sistemi basati su RedHat (es. CentOS). Permette di installare, aggiornare e rimuovere software. È stato sostituito da `dnf` nelle versioni più recenti.
dnf Nuovo gestore pacchetti Successore di `yum` su Fedora, CentOS 8+ e RHEL 8+. Offre miglior gestione delle dipendenze e prestazioni migliori. Es. `dnf install nomepacchetto` installa un software dal repository.
brew Pacchetti su macOS Gestore di pacchetti per macOS (e Linux). Usato per installare software open source. Es. `brew install nomepacchetto` scarica e installa in modo semplice senza modificare il sistema principale.
git Versionamento codice Sistema di controllo di versione distribuito. Permette di salvare lo storico del codice, collaborare con altri sviluppatori e gestire rami di sviluppo. Esempio: `git clone`, `git commit`, `git push`, `git pull`.
git clone Scarica repository Copia un repository remoto (es. GitHub) in locale. Esempio: `git clone https://github.com/utente/progetto.git`. Crea una copia completa del codice e della sua storia.
git status Stato modifiche Mostra i file modificati, aggiunti o rimossi rispetto all'ultimo commit. Aiuta a capire cosa c'è da salvare, inviare o ignorare prima di un commit.
git commit Salva modifiche Registra le modifiche al repository. Usato dopo `git add`, crea uno snapshot con un messaggio. Esempio: `git commit -m "Aggiunto file"`.
git push Invia modifiche Invia i commit locali al repository remoto. Serve per condividere le modifiche con altri sviluppatori o per salvare il lavoro su server remoti come GitHub.
git pull Scarica aggiornamenti Recupera e unisce gli aggiornamenti dal repository remoto nel ramo corrente. Mantiene sincronizzato il progetto con i contributi degli altri sviluppatori.
uname Info sul sistema Mostra informazioni sul sistema operativo. L'opzione `-a` mostra dati completi: nome kernel, versione, architettura, ecc. Utile per script e debugging.
lsblk Visualizza dischi Elenca i dispositivi a blocchi (dischi e partizioni). È molto utile per identificare dove sono montati i dischi, chi è `/dev/sda`, ecc. Non mostra file, ma hardware di memorizzazione.
mount Monta dispositivi Associa un dispositivo (es. USB, partizione) a una directory nel filesystem. Esempio: `mount /dev/sdb1 /mnt/usb` collega il dispositivo alla cartella `/mnt/usb` per accedere ai file.
umount Smonta dispositivi Rimuove l'associazione tra un dispositivo e il filesystem. Necessario prima di scollegare fisicamente un disco o chiavetta USB per evitare perdita di dati.
shutdown Spegne il sistema Programma o esegue lo spegnimento del sistema. Esempio: `shutdown now` spegne subito; `shutdown -h +10` spegne dopo 10 minuti. Spesso usato su server o terminali remoti.
reboot Riavvia il sistema Riavvia immediatamente il sistema. È equivalente a `shutdown -r now`. Utile per applicare aggiornamenti o ripristinare lo stato del sistema dopo modifiche importanti.



Ovviamente non si andranno a usare tutti quanti in un singolo contesto, questi comandi si imparano con la pratica e il tempo nell'ambiente di linux.
